Understand and internalize the following CSV format for defining Special Function Register (SFR) structures in a hardware system, presented as a Markdown table for readability. This format is used for tasks like generating, validating, or modifying SFR definitions. The CSV includes these fields: Group Name, Group Repeat Count, Group Size, Subgroup Name, Subgroup Repeat Count, Subgroup Size, Register Name, Offset, Bit Field, Bit Range, Default Value, Access Type, Testable, Description. Here are the rules and structure:

1. **Structure**:
   - Groups are defined with a name, repeat count (integer), and size (hex, e.g., 0x40).
   - Subgroups within groups have their own name, repeat count, and size.
   - Registers can belong to subgroups or exist standalone within groups or between groups for realistic designs.
   - Registers include bit fields with a name, bit range (e.g., [7:0]), default value (hex), and access type.

2. **Offset Management**:
   - Offsets (hex) must be unique, calculated based on group and subgroup sizes/repeat counts.
   - Groups start sequentially, with gaps treated as reserved (no need to define every address).
   - Specific groups may start at a fixed address (e.g., 0x100), with padding calculated automatically to align with prior group endings.

3. **Access Types**:
   - Support a variety of access types from the UVM reference[](https://verificationacademy.com/verification-methodology-reference/uvm/docs_1.1a/html/files/reg/uvm_reg_field-svh.html): RW, RO, WO, W1C, W0C, RC, RS, WC, WRC, WS, W1T, W0T, WO1, W1S, W0S, WRS, W1CRS, W0SRC.

4. **Testable Column**:
   - The Testable column uses `Y` for fields testable via read/write, reset value checks, or verifiable behavior (e.g., RW, RO, RC), and `N` for non-testable fields (e.g., WO, WO1 without read feedback).

5. **Simplification**:
   - Use single-row definitions combining group, subgroup, and register details where possible.
   - Leave fields blank for inherited values (e.g., Group Name for subgroups/registers) to reduce redundancy while maintaining clarity.
   - Provide human-readable descriptions for registers and bit fields.

6. **Usage**:
   - This format supports tasks like generating SFR definitions, validating offsets for uniqueness, modifying existing definitions, or calculating address ranges.
   - Ensure compatibility with CSV parsers and Markdown table rendering for web display.

   - 특정 그룹(예: IOGroup)은 고정 주소(예: 0x100)에서 시작할 수 있으며, 이전 그룹의 종료 주소에 맞춰 패딩이 자동 계산됩니다.

3. **접근 타입**:

4. **Testable 컬럼**:
   - Testable 컬럼은 읽기/쓰기, 기본값 검증, 또는 검증 가능한 동작(RW, RO, RC 등)의 경우 `Y`, 읽기 피드백이 없는 경우(WO, WO1 등) `N`을 사용합니다.
